# 已知问题与版本规划

## 已知问题

### 1. CLI 终端最小化后点击跳转无效

点击会话卡片跳转时，如果目标 CLI 终端窗口处于最小化状态，Windows 任务栏会显示窗口被选中，但窗口无法真正恢复到前台。这是 Windows 前台激活限制导致的，暂时没有稳定的解决方案。

### 2. VSCode 重启后 JSONL 匹配异常

关闭 VSCode 并在次日重新打开后，JSONL 文件的匹配规则可能出现偏差，导致会话状态读取不准确。问题与 JSONL 文件的 session ID 映射机制有关，在跨天重启的场景下尚未完全可靠。

## 未来版本规划

### VSCode 管道通信精准跳转

当前点击会话卡片只能激活 VSCode 的顶层窗口，无法定位到具体的 Claude Code 会话所在的终端面板。当同一个 VSCode 窗口内运行多个 Claude Code 会话时，用户仍需手动切换到目标终端。

计划通过 VSCode Extension 建立管道通信（Named Pipe / IPC），由扩展在 VSCode 内部根据 session ID 或 PID 精确定位到对应的终端面板并自动聚焦，实现从监控器到目标会话的一键直达。
